### YUV_Cleaner.bat 原有逻辑问题分析

1. **清理时机依赖间接信号**：bat 通过轮询检测**下一个** chunk 的 `_bl.yuv` 是否存在来决定删除当前 chunk 的 YUV 文件。虽然在顺序执行中功能正确，但这是间接判断——实际安全点应是当前 chunk 的 Vesmux 完成后（即所有 YUV 消费者都已结束）。
2. **轮询机制脆弱**：300 秒固定轮询间隔需要用户手动调整，且 bat 作为独立进程运行，与 Python 工作流缺乏同步，存在竞态风险。
3. **倒数第二个 chunk 的计算逻辑**：bat 中 `calculateFinalNumber` 循环在超过 `extraNumber` 后减去了 `intervalLength*2`（应减 `intervalLength`），导致 `finalNumber` 实际指向倒数第三个编号而非倒数第二个——不过这取决于对"倒数第二"的定义和实际 chunk 边界。

## 阶段详解

### 阶段 0：初始化与校验

| 步骤       | 类/函数                                   | 说明                                                         |
| ---------- | ----------------------------------------- | ------------------------------------------------------------ |
| 参数解析   | `create_parser()` → `Config`              | 解析命令行参数（输入文件、帧范围、chunk 大小、GOP、编码器类型等） |
| Chunk 划分 | `Config.get_chunks()`                     | 按 `chunk` 大小切分 `[start, end]`；若末尾 chunk < 2×GOP 则并入前一个；首 chunk 标记 `concatenate='false'` |
| 校验       | `sanity_check_dee`, `sanity_check_ffmpeg` | 调用 DEE 和 ffmpeg 检查可执行性                              |
| MXF 解析   | `parse_mxf()`                             | 仅 MXF/MXF_sidecar 输入时执行，提取索引 `.json`（及 `.xml`） |

### 阶段 1：逐 Chunk 编码循环（核心流水线）

**对每个 chunk 依次执行以下 6+1 步**（以 chunk `[0, 119]` 为例）：

```
┌─────────────────────────────────────────────────────────────────────┐
│  Step 1   Generate_bl_yuv     夹层 → BL 原始 YUV + manifest       │
│           产出: {ts}_0_119_bl.yuv, {ts}_0_119_bl_manifest.xml      │
│                                                                     │
│  Step 2   Encode_yuv(bl)      BL YUV → BL HEVC 码流               │
│           读取: _bl.yuv + _bl_manifest.xml                          │
│           产出: {ts}_0_119_bl.hevc                                  │
│                                                                     │
│  Step 3   Decode_bl_hevc      BL HEVC → 解码后 BL YUV (ffmpeg)    │
│           读取: _bl.hevc                                            │
│           产出: {ts}_0_119_bl_decoded.yuv                           │
│                                                                     │
│  Step 4   Generate_el_yuv     夹层 + 解码BL → EL YUV + RPU        │
│           读取: _bl_decoded.yuv + _bl_manifest.xml                  │
│           产出: {ts}_0_119_el.yuv, _el.rpu, _el_manifest.xml       │
│                                                                     │
│  Step 5   Encode_yuv(el)      EL YUV → EL HEVC 码流               │
│           读取: _el.yuv + _el_manifest.xml                          │
│           产出: {ts}_0_119_el.hevc                                  │
│                                                                     │
│  Step 6   Vesmux              EL RPU + EL HEVC → 混合 EL 码流     │
```
